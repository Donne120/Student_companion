The big picture

One platform, four moving parts.

A web app in the browser, a Python service in the cloud, a knowledge base staff control, and the AI models that write the answers.

AALU Student Companion02
The codebase

Two independent halves.

The frontend and backend are separate deployments. They meet at exactly one place: a single API URL.

Frontend — what students see

TypeScript + React, built with Vite, hosted on Vercel.

  • Chat UI with live, streaming answers
  • Firebase login — ALU emails only
  • History saved in the browser (localStorage)

Backend — the brain

Python + FastAPI, packaged in Docker, hosted on a Hugging Face Space.

  • Retrieves knowledge from a vector database
  • Orchestrates Claude, Groq, Tavily
  • Stateless — cheap and scalable

The core flow

A question's journey — in seven steps.

SEND
Only the last 10 turns are sent — caps cost & latency.
RETRIEVE
Two searches merged so the answer is grounded in real facts.
GENERATE
Context is cached — repeat asks cost up to 10× less.
STREAM
Words pushed live via SSE — first words in 1–2 seconds.
AALU Student Companion04
The heart of the system

It searches two knowledge bases, then merges them.

A question isn't answered from one source. The system runs a meaning-based search and a curated keyword search in parallel, then blends the best of both.

AALU Student Companion05
Why it doesn't make things up

Grounded generation — the guardrail.

The merged facts become mandatory context. Claude is instructed to answer only from them — never from memory.

The instruction to the model

"Answer only from the context provided. If a fact isn't there, say so and point to the official source." Style is layered on top — but the truth always wins.

  • No invented dates, fees, names, or course codes
  • Every answer carries its source citations
  • Voice is the delivery — facts are the substance

Built-in cost discipline

The system prompt and retrieved context are sent with prompt caching — stored by the model for 5 minutes.

  • Repeat questions cost up to 10× less
  • Only the last 10 messages are ever sent
  • A local 5-minute cache skips duplicate searches

AALU Student Companion08
Under the hood

The technology, end to end.

Proven, production-grade tools — each chosen for one job in the chain.

Frontend

React + TypeScript, built with Vite, hosted on Vercel. Firebase for ALU-email login.

Backend

Python + FastAPI in a Docker container on a Hugging Face Space. Streams over SSE.

Retrieval

ChromaDB vector store + all-MiniLM-L6-v2 embeddings, merged with a curated JSON brain.

Intelligence

Claude (Anthropic) writes answers; Groq · Llama is the free fallback engine.

Knowledge ops

Google Sheet + Apps Script feed the vector store. Polls every 15 min + instant webhook.

Live data

Tavily web search powers the opportunities feed, cached to respect free-tier limits.

  1. 1 · Title
    "This is the ALU Student Companion — an AI assistant that answers any student, any time, using the university's real knowledge. I'll walk you through how a single question travels through the system."
  2. 2 · Big picture
    "Four parts: a web app the student uses, a Python service in the cloud, a knowledge base our staff control, and the AI models that write the words. Everything hangs off these four."
  3. 3 · Two halves
    "The frontend and backend are fully separate — different languages, different hosts. They meet at one API URL. The backend is stateless, which keeps it cheap and easy to scale."
  4. 4 · The journey (CENTERPIECE)
    "Follow one question. The student types it; the frontend sends only the last ten messages to save cost. The backend retrieves matching knowledge, Claude writes an answer grounded in it, and it streams back word by word — first words on screen in one to two seconds."
  5. 5 · Hybrid retrieval (THE KEY IDEA)
    "Here's what makes it accurate. We don't search one place — we search two. A semantic search finds facts by meaning using vector embeddings; a curated brain finds exact question-and-answer matches by keyword. We merge the best of both and hand them to the AI."
  6. 6 · The guardrail
    "Those merged facts are mandatory. The model is told: answer only from this context; if it's not here, say so and point to the official source. That's why it doesn't hallucinate. And caching keeps repeat questions up to ten times cheaper."
  7. 7 · Knowledge pipeline
    "The big idea for sustainability: staff edit a Google Sheet — no code. A script publishes it, the backend pulls it in every 15 minutes, and the bot's knowledge updates in minutes. Uploaded documents feed the same store."
  8. 8 · Resilience
    "If the main model is down, a free fallback model answers with the same context. If that fails, a graceful message points to support. Students almost never hit a dead end."
  9. 9 · Beyond chat
    "It's more than Q&A: an application-essay coach, a live opportunities feed, and an admin analytics dashboard."
  10. 10 · Tech stack
    "Briefly, the tools: React on Vercel; FastAPI in Docker on Hugging Face; ChromaDB and MiniLM embeddings for retrieval; Claude and Groq for intelligence; Firebase, Google Sheets, and Tavily around the edges. All proven, production-grade."
  11. 11 · By the numbers / 12 · Three ideas / 13 · Thank you
